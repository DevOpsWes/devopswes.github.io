## Writing a New Blog Post

1. Create a file in `_posts/` named exactly: `YYYY-MM-DD-your-post-title.md`
2. Add this front matter at the top:

```yaml
---
title: "Your Post Title"
date: 2026-01-01
tags: [AI, DevOps, Automation, Virtual-Desktop-Environments]
excerpt: "One sentence that appears in post cards and SEO meta description."
---
```

3. Write your content in Markdown below the `---`
4. Commit and push → site rebuilds in ~1 minute

**Supported tags:** `AI`, `DevOps`, `Cloud`, `Automation`, `Virtual-Desktop-Environments` (add new ones freely — they auto-appear on `/tags`)

**Code blocks** use Rouge syntax highlighting:

````markdown
```python
def hello():
    print("Hello, world!")
```
````

## Copilot Agents

Two Copilot instruction files in `.github/instructions/` handle blog content quality. Both are scoped to `_posts/**` and load automatically when working on post files.

### Blog Writing Agent

**File:** `.github/instructions/blog-writing-agent.instructions.md`

Guides Copilot through a 5-step fact-first drafting process designed to produce posts that read as genuinely written by Wesly, not generated by AI:

1. Extract raw facts (exact tool names, versions, durations, failures) before writing any prose
2. Commit to a post type: story, opinion, or how-to
3. Write a rough draft from those facts
4. Run an anti-generic rewrite pass ("could any DevOps blogger have written this sentence?")
5. Self-check against a publishing checklist

The file also includes a palette of Wesly's real micro-patterns (punchy one-liners, rhetorical colons, honest admissions, reflective endings) and before/after rewrite examples. These are options to draw from, not a template to satisfy.

**Use it by:** asking Copilot to write or draft a blog post in `_posts/`. The instruction file loads automatically.

### AI Detection Agent

**File:** `.github/instructions/ai-detection-agent.instructions.md`

A review rubric that scans a draft for AI-generated patterns and reports each one with the flagged line, a reason, and a suggested rewrite in Wesly's voice.

Covers 9 pattern categories:

| Pattern | What it catches |
|---|---|
| Throat-clearing intros | Posts that open with context-setting before the actual situation |
| Balanced triads | Groups of 3 perfectly parallel items; uniform bullet lists |
| Generic transitions | "Furthermore", "Moreover", "Additionally", "As a result" as sentence openers |
| Abstract claims | Insights with no tool name, version, environment, or incident attached |
| Uniform paragraph rhythm | Consecutive paragraphs of identical length — humans write unevenly |
| Too-clean causality | Frictionless problem-solution arcs with no dead ends or doubt |
| Summary endings | Final paragraphs that restate what the reader just read |
| Interchangeable prose | Sentences that could appear in any DevOps blog without modification |
| Banned phrases | Full list from `writing-style.instructions.md` |

Also checks for 5 missing human signals: friction/failure, named specifics, direct "you" address, varied sentence length, and at least one real opinion.

**Use it by:** asking Copilot to "review for AI patterns", "run the AI check", or "validate this post" on any draft in `_posts/`. It's explicitly invoked — not always-on.
